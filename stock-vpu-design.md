# 股票流动性深度指标（VPU）— 设计方案

> **版本**：v1.5
> **日期**：2026-03-08
> **状态**：已确认

---

## 一、指标定义

**名称**：单位价差成交量（Volume Per Price Unit, VPU）

**含义**：推动股价变动一个最小计价单位（0.05元）所需的平均成交量。值越大，市场越"厚"，流动性越好；值越小，价格越容易被推动。

**同时提供成交额版本**（Amount Per Price Unit, APU），用于跨股票横向比较：

```
VPU = 成交量（手） / max(1, ceil(前复权价差 / 0.05))
APU = 成交额（元） / max(1, ceil(未复权价差 / 0.05))
```

> **复权策略**：VPU 使用前复权价格计算价差（保证跨除权日的量价连续性）；APU 使用未复权价格计算价差（保证成交额与价格尺度绝对一致）。两者分开处理，互不影响。

## 三、计算流程

### 3.1 输入验证与数据采集

```
输入：股票代码、起止日期（默认最近一个月）

验证：股票代码须匹配 A 股格式（正则校验）
  - 主板：600xxx, 000xxx, 001xxx, 002xxx, 003xxx
  - 创业板：300xxx, 301xxx
  - 科创板：688xxx
  - 可选 sh/sz 前缀
  无效代码在 CLI 和 Streamlit 入口处直接拦截，避免无效 API 调用。

输出：该股票在指定时间范围内的5分钟K线数据（OHLCV + 成交额）
```

### 3.2 数据清洗（逐交易日）

每个交易日理论上有 **48个** 5分钟单元（240分钟 / 5分钟）。

**剔除规则**（按优先级执行）：

| 规则 | 条件 | 原因 |
|------|------|------|
| R1. 涨跌停日 | 当日最高价或最低价触及涨跌停价：`(最高价-昨收)/昨收 > 9.8%` 或 `(最低价-昨收)/昨收 < -9.8%`（科创/创业板阈值为 19.8%；ST 股阈值为 4.8%） | 成交行为异常，流动性指标无意义 |
| R2. 首尾单元 | 9:30-9:35 和 14:55-15:00 | 集合竞价/尾盘抢单导致数据失真 |
| R3. 零价差单元 | 最高价 == 最低价 | 除零，且横盘无价格发现意义 |
| R4. 极低振幅单元 | 价差 < 0.03元（不足3个tick） | 波动过小导致 VPU 异常膨胀，数据噪声大于信号 |

剔除 R2 后剩余 **46个单元**，再剔除 R3、R4 后得到 **N个有效单元**。

> **特殊情况**：若某日有效单元数 < 10，该日标记为"数据不足"，不参与指标计算。

### 3.3 单元指标计算

对每个有效5分钟单元，采用**离散化阶梯计算**避免小数放大效应（向量化实现，使用 `np.ceil().clip(lower=1)`）：

```
价差 = 最高价 - 最低价

# 离散化：不足1个单位算1个，超过则向上取整（向量化）
价差单位数 = clip(ceil(价差 / 0.05), min=1)

# VPU（手数）使用复权价对应的价差计算（保持量价物理意义）
VPU_i = 成交量 / 价差单位数

# APU（金额）强制使用【未复权价差】计算（保持成交额与价格尺度的绝对一致）
APU_i = 成交额 / max(1, ceil(未复权价差 / 0.05))
```

> **注**：复权导致的结构性偏差只影响涉及金额绝对值的计算。分离前复权（VPU）和未复权（APU）是保持横向/纵向指标一致性的必须手段。

### 3.4 截尾均值（日级聚合）

将当日 N 个有效单元的 VPU 值排序：

```
截尾比例 = 25%（上下各截掉25%）
截掉数量 = floor(N * 0.25)

截掉最大的 floor(N*0.25) 个
截掉最小的 floor(N*0.25) 个
剩余 ≈ 50% 的中间值取算术平均 → 当日 VPU

APU 同理处理
```

> **为什么25%而非原方案的33%**：保留50%的数据在去除异常值和保留统计意义之间取得平衡。原方案67%截尾过于激进，容易丢失有效信息。

### 3.5 方向性指标（可选扩展）

将有效5分钟单元按涨跌方向分组，分别计算 VPU，用于识别上方抛压与下方支撑的不对称性：

```
VPU_Up：仅计算 收盘价 > 开盘价 的单元（上涨单元）
VPU_Down：仅计算 收盘价 < 开盘价 的单元（下跌单元）
收盘价 == 开盘价 的单元不参与方向性计算（归入总体 VPU）
```

**解读**：
- VPU_Up >> VPU_Down → 上涨需要更多成交量推动，上方抛压重
- VPU_Up << VPU_Down → 下跌需要更多成交量推动，下方支撑强
- VPU_Up ≈ VPU_Down → 多空力量均衡

> 方向性指标同样使用截尾均值聚合（3.4节），APU_Up / APU_Down 同理。

### 3.6 移动平均（平滑处理）

在日级指标基础上，叠加移动平均线以识别趋势：

| 均线 | 用途 |
|------|------|
| **5日均线** | 捕捉短期流动性变化 |
| **10日均线** | 识别中期趋势 |

```
VPU_MA5[t]  = mean(VPU[t-4 : t])
VPU_MA10[t] = mean(VPU[t-9 : t])
APU 同理
```

## 四、异常状态标注

在最终图表上标注以下事件日（辅助解读指标突变）：

| 事件 | 数据来源 | 标注方式 |
|------|----------|----------|
| 涨停/跌停 | K线数据判断 | 红/绿三角标记 |
| 除权除息日 | AKShare 复权因子 | 垂直虚线 |
| 财报发布日 | 可选，手动输入或API | 垂直虚线 + 文字 |

> 除权除息日需特别注意：除权后股价跳变，前后的绝对价差指标不可直接比较。**VPU 使用前复权价格**计算以保持连续性；**APU 使用未复权价格**计算以保持成交额与价格尺度一致（详见3.3节）。

## 五、可视化输出

### 5.1 核心视图：K线深度联动图 (Candlestick + VPU)

**视觉设计（双 Grid 结构）**：
- **上方 Grid (60% 高度)**：
  - **Candlestick**：展示每日开/高/低/收（阳线 `#ef5350`, 阴线 `#26a69a`）。
  - **MA5/MA10**：叠加价格移动平均线。
- **下方 Grid (30% 高度)**：
  - **VPU 零轴对立柱**：`VPU_Up` 向上（红色渐变），`VPU_Down` 向下（绿色渐变）。
  - **样式优化**：柱体增加圆角及半透明渐变。

**交互与联动**：
- **轴联动 (Sync)**：通过 `axisPointer.link` 绑定两个 Grid 的 X 轴，缩放、平移其中一个，另一个自动同步。
- **高级 Tooltip**：悬停时同时显示 OHLC 价格、涨跌幅及对应的 VPU 抛压/支撑数值。
- **DataZoom**：放置于最底部，同时控制上下两个区域的视图范围。

## 六、系统架构与输入输出

### 6.1 技术架构

本项目采用 **双引擎渲染** 与 **高效缓存** 架构：

1. **表现层 (app.py)**：基于 **Streamlit**。仅负责 UI 布局、状态路由与输入响应。
2. **呈现层 (visualizer.py)**：封装所有图表渲染逻辑。
   - 包含 `render_chart` 和 `render_apu_chart` 生成 ECharts 配置。
   - 包含 `wrap_js_code` 工具函数，用于将 JavaScript 字符串包装为可执行代码（实现 Tooltip 的动态正值显示）。
3. **性能层 (@st.cache_data)**：数据采集 (`akshare`) 与核心计算 (`calculate_vpu`) 逻辑被封装在带缓存的函数中。
   - **TTL = 3600s**：相同参数的请求在一小时内直接返回缓存，极大减少 API 访问压力。
   - **参数序列化**：通过对 `Config` 对象的哈希处理，确保参数变动能实时触发重算。

### 6.2 输入界面 (Streamlit 网页仪表盘)

为了方便在使用初期高频调试各项指标参数（如截尾比例、单位价差），推荐使用 **Streamlit** 构建一个轻量级的网页交互界面。

**左侧边栏（控制面板）**：
- **基础设置**：
  - 股票代码输入框（如 `600519`）
  - 时间范围选择器（日期区间：起 - 止）
- **核心参数滑动条（实时重绘图表）**：
  - `PRICE_UNIT`（最小价差单位）：滑动调节，例如 `0.01` ~ `0.10`（默认 0.05）
  - `TRIM_RATIO`（截尾比例）：滑动调节，例如 `0%` ~ `40%`（默认 25%）
  - `MIN_PRICE_SPREAD`（最小有效价差）：滑动调节，例如 `0.01` ~ `0.05`（默认 0.03）
- **执行按钮**：
  - `获取数据并计算`（带加载动画）
  - `导出数据为 CSV`

**右侧主区域（结果展示）**：
- **核心数据看板**：显示该股票在选中时间段内的平均 VPU、平均 VPU_Up、平均 VPU_Down，以及当前的流动性状态诊断（如"上方抛压重"、"支撑极强"）。
- **交互式图表区**：嵌入上述 5.1 节定义的 ECharts（或 Plotly）零轴对立图。

### 6.2 命令行接口（CLI，作为备选/自动化入口）

```
- 股票代码：如 "600519"（贵州茅台）
- 时间范围：默认最近一个自然月，可自定义起止日期
```

### 6.3 输出规格

```
- 终端/Notebook 中显示图表
- 导出 PNG 图片
- 导出 CSV 原始数据（日期、VPU、VPU_Up、VPU_Down、APU、MA5、MA10、事件标注）
```

## 九、可配置参数汇总

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `PRICE_UNIT` | 0.05 | 最小价差单位（元） |
| `TRIM_RATIO` | 0.25 | 截尾比例（上下各截掉25%） |
| `MIN_VALID_UNITS` | 10 | 每日最少有效单元数，不足则该日作废 |
| `MIN_PRICE_SPREAD` | 0.03 | 最小有效价差（元），低于此值的单元剔除（R4规则） |
| `MA_PERIODS` | [5, 10] | 移动平均周期 |
| `SKIP_FIRST_LAST` | True | 是否剔除首尾5分钟单元 |
| `ENABLE_DIRECTION` | True | 是否计算方向性指标（VPU_Up / VPU_Down） |

> **复权策略为固定规则**（非可配置）：VPU 始终使用前复权价格，APU 始终使用未复权价格。详见3.3节。
